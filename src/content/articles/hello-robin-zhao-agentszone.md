## 我为什么来特区

读马工文章最久的一篇，除了 [FDE 批判](forward-deployed-engineer)，其实是他对 **制度与 Harness** 的坚持：规格定义输入，自动化验证检查输出，规约本身也要迭代。

特区三个讨论概念，我都踩过坑：

| 特区概念 | 我的现场版本 |
|----------|--------------|
| **自包含文档** | `AGENTS.md` + handoff + BAM；没有这些，下一次 Cursor 会话会重写同样的错 |
| **文档测试** | W8 五分钟演示脚本先写进文档，再让客户 IT 口述演练 |
| **全能小团队** | 铁三角：顾问（BAM/Steering）+ Owner（拍板）+ 我在现场的 PoC/Runbook |

所以我来报到，是想找 **同样把 artifact 当交付物** 的人对线，而不是找下一个「模型更强了」的借口。

## 我在做什么（三条线）

### 1. FDE · 90 天驻场（学科，不是外包）

我把 FDE 定义成 **企业 AI 时代合伙人**，不是 IT 系统工程师。成功标准写死在节点上：

- **W4**：范围 + KPI + 集成 ≤2，Steering 书面确认  
- **W8**：可演示 PoC，人审链在演示里**必须点给用户看**  
- **W12**：bundle + Runbook + 客户 IT 能复述主流程  

不验收成交额、不验收融资——那是业务结果。我验收 **可审查、可续谈、可交接**。

### 2. Ontology-Robin · 语义链

Excel → 图谱 → Agent 四步推演。对象—关系—动作—证据，金融零售等场景有完整 W8 脚本。这里和马工说的 Harness 黄金集是同一类东西：**别让 Agent 静默过关**。

### 3. IP 评估与转化（场景 D）

高校、园区、河套类型的 **专利导航 → 评估 → 专利库 → 挂牌/意向 → 技术转移**。工程上要落实经纪人的 Deal/HALT 闸门——`licenseTrimmable=no` 时界面要拦，不是事后甩锅给法务。

## 和马工 FDE 文的关系

我同意他很多批判：没有产品的「平台 + 常驻工程师」是在卖人天；Palantir 模式不能无脑抄。

但我仍选择在前线干活，因为在中国现场，客户缺的不是第 N 个 Copilot 入口，缺的是：

1. **有人帮他们把范围冻住**（否则 Agent 只会加速范围蔓延）  
2. **有人把验收写成脚本**（否则 W12 只有代码没有签收）  
3. **有人敢在演示里说「这里必须人审」**（否则合规在 PPT 里，事故在上线后）

如果这也算 FDE，那我认。如果不算，你们可以帮我起一个更好的中文名——我无所谓名字，在乎 **W12 能不能签收**。
