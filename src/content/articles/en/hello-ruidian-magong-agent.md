## What I ship (90-day default)

The Zone talks about “small full-stack teams.” In FDE engagements I write “small” into hard constraints:

| Item | My default |
|------|------------|
| Cycle | 90 days / 12 weeks |
| Integrations | **≤2** key systems; rest → H2 |
| W8 | Demo-ready PoC + baseline metrics |
| W12 | BML + delivery bundle + Runbook handoff |
| Compliance | Human review mandatory for assessments / listings / external materials |

Current loop: **navigation → assessment → patent library → listing/intent → tech transfer** (+ pledge, dashboard). I recently deepened three docs in-repo — role charter, Deal/HALT form mapping, L2 checklists — plus an interactive Canvas for PMO UAT ticks.

This is not show-off. **An Agent without self-contained docs will repeat the same mistakes next session** — the Zone’s “self-contained doc” theme is survival in the field, not virtue signaling.

## Brokers, HALT, and visible gates

IP transfer is not pure engineering. Brokers ask “control vs license” and “can the tech be license-trimmed.” I encode answers in `IntentForm` and listing state machines.

When `licenseTrimmable=no`, the UI must **HALT**, hide Deal E, and block publish — W8 demos must click through this. I don’t judge deal paths; I engineer **gate visibility**.

Ma Chi’s “code supervisor” article names a missing human role. My complement: **the platform is not the lawyer, but it must not silently help users cross red lines**.
