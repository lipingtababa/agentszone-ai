# Agents 特区

**一起探索怎么把软件工程和 AI 结合，让我们最终失业。**

**[agentszone.ai](https://agentszone.ai)**

## 网站上有什么

### 文章

社区成员写的实践经验。怎么在团队里用 Agent、怎么管质量、怎么搭流程——都是真干过的人在写。目前有 12 篇中文、7 篇英文。

**你可以贡献：** 写你自己的实践。复盘、踩坑、方法论都行。Fork repo，加一篇 markdown，提 PR。详见 [CONTRIBUTING.md](CONTRIBUTING.md)。

### 作者

每个写过文章的人都有一个 profile 页。目前 11 位作者。

**你可以贡献：** 写了文章就自动成为作者。创建 `src/content/authors/<你的slug>.md`，填上你的信息。

### 讨论

从社区对话中提炼出来的关键概念，目前有三个：

- **自包含文档** — Agent 没有团队记忆，规格说明必须完全自包含，不能有隐含知识
- **文档测试** — 写代码之前先让 AI 走一遍用户故事，在文档阶段就发现漏洞
- **全能小团队** — AI 压缩了传统角色，3-5 人团队可以替代 10-15 人团队

每个讨论关联到 Playbook 的对应章节和相关文章。

**你可以贡献：** 群里讨论出了值得沉淀的概念？写成一篇讨论，关联到相关文章和 Playbook 章节。

### 活动

每周线上研讨会的记录，目前 15 期。话题从 BMaD 到端到端测试到土木工程类比都有。

**你可以贡献：** 参加过研讨会？帮忙补充活动记录。或者报名做下一期分享。

### 海报生成器

给活动做宣传海报的工具，直接在网站上用。

## 所有内容都是双语的

中文是默认语言，英文版是完整镜像。写中文就行，英文版我们会补上。当然你也可以直接写双语。

## 如何提交

让你的 AI Coding Agent fork 这个 repo，写好内容，提个 PR。技术细节见 [CONTRIBUTING.md](CONTRIBUTING.md)——它是写给你的 Agent 看的。
